 0ctx Zero context loss. For everyone who uses AI. Domain 0ctx.com Version Strategy v2.0 Date February 2026 Status Internal — Confidential 1. The Problem AI tools are stateless. Your life is not. Every time you open a new AI tool — or even a new conversation in the same tool — it knows nothing about you, what you&apos;re working on, the decisions you&apos;ve already made, or the constraints you&apos;re working within. You re-explain from scratch. Every single time. This is not a developer problem. It is a universal problem. It affects everyone who uses AI to think, work, create, and decide. Who feels this What they re-explain A lawyer working a case Background, parties, jurisdiction, strategy — across every AI tool they touch A designer briefing AI tools Brand tone, constraints, client preferences, what&apos;s been ruled out and why A researcher synthesising ideas The hypothesis, sources reviewed, threads being followed, dead ends hit A founder using multiple AIs Company context, stage, goals, priorities — pasted manually into every session A student writing a thesis The argument, the literature, the structure, the voice — reset every chat A developer building a product Architecture decisions, constraints, the why behind every technical choice The root cause is the same everywhere: AI tools have no shared memory of your active work context. Every tool switch, every new tab, every new session — the slate is wiped. Your thinking doesn&apos;t reset. The tools do. 2. What 0ctx Is 0ctx is the context layer for everyone who uses AI. A persistent, local-first, graph-based engine that holds the living state of your thinking — and makes it available to every AI tool you use, instantly. What the OS does for applications, 0ctx does for AI context. The experience: you have been working through a problem in Claude — a legal strategy, a design brief, a research question, a product architecture. You open a different tool to continue. That tool already knows. The decisions you made. The constraints you&apos;re working within. The open questions still in flight. You say nothing. You re-explain nothing. You just continue. What 0ctx is What 0ctx is not Active working memory for your current context Long-term personal memory or user profiling A reasoning graph — decisions and their relationships A flat notepad or chat log A continuity layer across any AI tool switch A logging or session replay tool A handoff format between people and tools A documentation generator you have to maintain 3. Who It Is For 0ctx is for anyone whose thinking spans more than one AI tool or more than one session. That is, almost everyone using AI seriously today. 3.1 The Beachhead — Technical Users Developers, engineers, and technical founders feel the pain most acutely and will configure a tool themselves. They are the beachhead — early adopters who adopt because the product genuinely solves a daily problem, and who influence tool choices for their teams. 3.2 The Broader Market — Knowledge Workers The same problem exists for anyone doing sustained, context-rich work with AI: lawyers, researchers, writers, designers, consultants, analysts, students, founders. The work context is different. The frustration is identical. The solution is the same. 3.3 Teams When work changes hands, context evaporates. The new person reads the thread, the doc, the ticket — and reconstructs a rough mental model. Lossy, slow, error-prone. 0ctx makes team handoffs structural. The reasoning travels with the work, not in someone&apos;s head. 4. The Knowledge Graph The graph is the core data model. Not a flat list of notes. Not a chat history. A living, typed, relational graph of your active thinking. Context is inherently relational. Decisions depend on constraints. Assumptions underpin choices. Open questions connect to the decisions that will resolve them. A flat list loses the because — the logic underneath — and the because is everything. 4.1 Node Types — Universal, Not Developer-Specific background — the essential context about who you are, what you&apos;re working on, and why decision — a choice made, with the reasoning behind it constraint — a hard limit or requirement that shapes the work goal — what you are trying to achieve in this context assumption — something believed to be true that has not been verified open_question — an unresolved issue still in flight artifact — a canonical piece of content, document, or reference 4.2 Edge Types caused_by — this decision followed from that constraint constrains — this requirement limits that decision space supersedes — this replaces a previous decision or assumption depends_on — this artifact or goal depends on that decision contradicts — this constraint conflicts with that assumption 4.3 What the Graph Unlocks Impact tracing — change one decision, see everything downstream that may need revisiting Structured handoff — hand off a subgraph, not a summary someone had to write Conflict detection — contradicts edges surface implicit contradictions before they become problems Smart context for AI — instead of dumping everything, give the AI the relevant subgraph for the current task 5. Architecture 5.1 The Persistent Daemon 0ctx runs as a persistent local daemon. One process, always running in the background. Every AI tool that connects to it is just a client of the same shared state. This is what makes cross-tool continuity possible — not spawning a new process per tool, but one persistent home for your context. 5.2 MCP as the Integration Layer 0ctx exposes all its capabilities as MCP tools. Any tool that speaks MCP — Claude, Cursor, Claude Code, and anything built on the protocol going forward — connects to 0ctx and gets a full set of tools it can call naturally mid-conversation. One MCP server, universal compatibility. For non-technical users, the integration surface will be a browser extension and simple one-click connections — no config file editing required. The MCP layer is the technical backbone; the product surface adapts to the user. 5.3 Project and Context Isolation Everything belongs to a named context — a project, a case, a brief, a research thread. Contexts are fully isolated. Your legal case context never bleeds into your personal project context. Within a context, threads allow parallel workstreams — two approaches being explored simultaneously, each with its own subgraph. 5.4 Auto-Extraction The graph builds itself. As you work — talking with an AI, thinking through a problem — 0ctx watches the conversation and extracts the signal automatically. When a decision is made, it becomes a decision node. When a constraint is mentioned, it becomes a constraint node with an edge to whatever it constrains. No manual saving. No remembering to log things. The context captures as you think. 5.5 Database SQLite locally. Zero infrastructure, single file, instant startup, works offline. The graph lives in SQLite on your machine. Cloud storage for the sync layer uses server-side infrastructure, but this is invisible to the user. The local file is always the source of truth. 6. Handoff Handoff is the product&apos;s most powerful surface. When work changes hands — between tools, between sessions, between people — 0ctx packages the relevant subgraph into a context packet: a structured, relevance-ranked slice of the graph with a natural language summary generated from it. The receiving tool or person gets both: the structured graph to query and the human-readable summary to read. No re-explanation. No reconstruction from a Slack thread or a doc someone had to write after the fact. The reasoning travels with the work. This applies equally to handing work from Claude to Cursor, from a senior team member to a junior, from yourself on Friday to yourself on Monday morning. 7. Checkpoints A checkpoint is a named snapshot of the context graph at a specific moment. You can rewind to any checkpoint — removing everything added after that point. This is essential for exploratory work: try an approach, decide it is not right, rewind to before you started it. Context branching extends this further: fork a context at a checkpoint to explore two approaches simultaneously, then merge the one that worked back into the main graph. The graph model handles branching naturally. 8. The UI The UI is a local web app served by the daemon — open a browser to localhost and it is there. For technical users this is sufficient. For everyone else, a browser extension makes the UI always accessible as a sidebar without thinking about localhost addresses. Graph view — the actual node-and-edge graph, not a list. Click any node to see its connections and edit it inline. Context switcher — jump between contexts instantly. Full isolation between them. Timeline — scroll back through how the graph evolved. Every checkpoint is a point on the timeline. Search — find any node by content, type, or relationship. Manual override — full control. Add nodes, edit content, delete things the AI got wrong, connect things manually. In the SaaS version, this UI lives at 0ctx.com — accessible from anywhere, with shared contexts visible to the whole team. 9. SaaS Layer The local daemon is always the core. SaaS is the layer that connects machines and people. It never replaces the local experience — it extends it. 9.1 What SaaS Unlocks Multi-machine sync — your context graph is current on every device you use. Team shared contexts — a decision made in one person&apos;s session is available in every teammate&apos;s tool. Web dashboard — your graph from any browser, shareable with teammates, with full history. Audit trail — every decision made during AI-assisted work, preserved. For compliance, onboarding, and institutional memory. 9.2 Trust and Encryption Context is sensitive. It contains reasoning, strategy, constraints, and half-formed ideas that people do not want stored in plaintext on a cloud server. 0ctx is end-to-end encrypted by default. The encryption key never leaves the user&apos;s machine. The cloud stores encrypted blobs it cannot read. Zero-knowledge is the default, not an enterprise add-on. 9.3 Tiers Tier Who It Is For Key Unlock Free Anyone, local only Full graph, MCP, UI — no limits, no cloud Pro · $12/mo Multi-device users Cloud sync, web dashboard, multi-device Team · $20/seat/mo Teams Shared contexts, team graph, access control Enterprise Regulated orgs On-premise, SSO, audit logs, zero-knowledge Free is genuinely useful, not crippled.
