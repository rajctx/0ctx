# Release

Release validation is script-driven.

## Dry-run validation

```bash
npm run release:validate -- -DryRun -AllowDirty
```

Validation steps:

1. typecheck
2. build
3. test
4. GA agent e2e
5. daily flow e2e
6. desktop smoke
7. nested git check

## Consolidated release report

```bash
npm run release:report
```

This runs the real release gates and writes:

- `releases/verification/release-readiness.json`

## Verification artifacts

Release verification reports are written to:

- `releases/verification/ga-agents.json`
- `releases/verification/daily-flow.json`
- `releases/verification/release-readiness.json`

These reports are generated by:

- `npm run release:e2e:ga`
- `npm run release:e2e:daily`

## Current release assumptions

GA validation is based on:

- Claude
- Factory / Droid
- Antigravity

Preview integrations are not part of the normal release gate.

## Packaging

Desktop packaging and smoke coverage use:

```bash
npm run desktop:smoke
cd desktop-app
npm run build:debug
```

## Before publishing

Minimum expected state:

1. `npm run build` passes
2. `npm run test` passes
3. `npm run release:e2e:ga` passes
4. `npm run release:e2e:daily` passes
5. `npm run desktop:smoke` passes
6. working tree is clean unless intentionally doing a dry-run with `-AllowDirty`
